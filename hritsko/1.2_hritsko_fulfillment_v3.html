Why this system exists
The bottleneck is not production — it is packaging and fulfillment.

Hritsko can make product faster than it can package and ship it. Without a defined system, that gap creates wrong shipments, untraceable errors, and a founder who is executing instead of selling. This document is the fix.

What it solves

Wrong product, wrong label, and shipments that can't be traced back to a batch if something goes wrong.

What it enables

Eric focused on growth. Leighanna assisting with ops—adding structure and visibility so the floor runs without daily firefighting.

Built for scale

Designed modularly. Start with Google Sheets. Add scan workflows, forms, and automation as volume grows.

What to watch at 30 days

QA failure rate dropping to zero. No mislabeled shipments. Leighanna able to run a pack day without founder involvement. Status visible without asking.

What to watch at 2x volume

Pack Log bottleneck (fix: Google Form). Batch running short mid-pack (fix: qty warnings). Status lag (fix: Apps Script auto-update). These are known and planned.

This system is designed to be adjusted. If a step doesn't fit reality, change it. The structure matters more than the exact format. The non-negotiables are: every order has an ID, every order links to a batch, nothing ships without a QA sign-off.

Leighanna — daily execution guide. Follow in sequence. Do not skip steps.
Active
01 · Core workflow
How every order moves
01
Order received
Log to ORDERS tab: Order ID · Customer · SKU · Qty · Date. Set status: PENDING.
02
Assign batch ID
Find the correct batch in BATCHES tab. Confirm QA Status = RELEASED. Enter the Batch ID into Column H on the order row.
Gate: batch not released = do not pack
03
Pack the order
Follow the packaging checklist. Every item must be checked before sealing. Update status to PACKING.
04
Log and stage for QA
Fill in the PACK LOG row completely. Sign initials. Move the package to the QA staging area — not the outbound pile.
Gate: no log entry = order does not move forward
05
QA release
Second person (not the packer) verifies. Signs initials. Sets status to READY. Moves package to outbound.
Gate: no QA sign-off = does not ship
06
Ship and close
Hand to carrier. Log Shipment ID + Tracking # in SHIPMENTS tab. Set Order status to SHIPPED.
Done — record closed
02 · Packaging checklist
Before sealing the package
Before touching product
  • Order ID and Batch ID are both filled in on the order row
  • Batch QA Status = RELEASED in Batches tab
  • Batch container label matches the Batch ID exactly
Product & label
  • Product matches order SKU — read the name, do not assume
  • Quantity counted twice
  • Label name matches product SKU
  • Shipping label shows correct customer name
Final
  • Package fully sealed, no gaps
  • Pack Log row filled in, initials signed
  • Package moved to QA staging area

Date:
03 · QA / release check
Before handing to carrier
Batch release — once per batch
  • Batch ID, date, product, and qty logged in Batches tab
  • Product visually inspected — no defects
  • Batches tab QA Status updated to RELEASED
  • Initials and release date logged
Order release — per order
  • Pack Log row fully filled in (all YES columns)
  • Physical check: correct product, label, sealed
  • Shipping label shows correct customer and address
  • Order status updated to READY
  • Tracking number logged in Shipments tab
If QA fails
  • Mark Pack Log QA Result = FAIL, enter reason
  • Return to packing area — do not delete the failed row
  • Repack → new Pack Log row → resubmit to QA

03 · Critical failure points
Where this breaks first
Skipping the pack log under pressure
No log entry = order does not ship. Non-negotiable from day one.
Packer self-checking QA
QA sign-off must always be a second person — even 30 seconds is enough to catch a label error.
Batch IDs not assigned before packing starts
Leighanna assigns Batch IDs each morning. Operators must never guess which batch they are pulling from.
Status columns lagging behind reality
Updating status is part of the step, not optional. Stale Dashboard = bad decisions.
No physical separation between QA and outbound
Two staging areas required. Same pile = orders bypass QA entirely.

05 · Version 2 — after 2 weeks
What to improve once it's running
Google Form for Pack Log — replaces manual Tab 3 entry. Operator submits from phone in 45 seconds. Writes directly to the sheet. Takes one hour to build.
Conditional formatting on Batches!F — Qty Remaining below 20 → amber, below 5 → red. Leighanna sees low stock before it becomes a floor problem.
QR codes on batch containers — encode the Batch ID, print free at qr-code-generator.com. Scan to verify instead of typing. Eliminates transcription errors.
Errors log tab — one row per QA FAIL. After two weeks, patterns emerge: same operator, same SKU, same time of day. First training signal.
